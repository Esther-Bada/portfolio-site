What I Didn't Expect From Building My First Real Website With AI

I didn't come into this trying to become a software developer.

My background is in education, understanding how people learn, how systems fail them, and how those gaps can be closed. So when I started building with AI tools, it wasn't about "learning to code" in the traditional sense. It was about figuring out how to actually build something real, from scratch, and get it live.

This project, my portfolio website, is the first time I've taken something from idea to a working, deployed system.

The Stack I Used

To get the site live, I used a simple but complete toolchain:

  • Claude Code: for building, editing, and debugging directly inside my project
  • VS Code: as my development environment
  • Live Server: for real-time preview while building
  • GitHub: for version control and tracking my work
  • Netlify: for deployment and hosting
  • Netlify Forms: to handle contact form submissions without a backend

None of these tools on their own are complex. What matters is how they connect.

Why I Chose These Tools

I chose Netlify because it removes the need for managing servers. I didn't need to think about infrastructure, I just needed a way to get my site live quickly and reliably.

Instead of setting up a full database, I used Netlify Forms. It handles form submissions automatically, which meant I didn't have to build a backend just to receive messages.

Claude Code became the most important tool in the process. It allowed me to describe problems in plain language, see how they connected to my actual files, and fix issues without getting stuck for hours.

What It Cost Me

One thing I paid attention to while building was cost.

  • VS Code: free
  • Live Server: free
  • GitHub: free (for public repositories)
  • Netlify: free tier (hosting and forms)
  • Claude: paid plan (but optional depending on usage)

It's possible to build and deploy a working website with almost no upfront cost. That changes how accessible building becomes.

The Hard Part: Deployment

The most challenging part of this project wasn't writing code, it was deployment.

I ran into repeated issues trying to connect my GitHub repository to Netlify. Even after configuring everything correctly, the deployed site kept returning a 404 error. The manual deployment worked, but the Git-integrated version didn't.

That experience taught me something important: building the site is only half the work. Getting it live, and understanding how the system behaves in a real environment, is where things become real.

What Changed For Me

Before this, I saw coding as something that required years of learning before you could build anything meaningful.

Now, I see it differently.

I can build real systems. I can deploy them. I can debug them. And I can do that without coming from a traditional software engineering background.

What matters is not just knowing syntax — it's knowing how to think, how to structure problems, and how to use the right tools effectively.

Where This Goes Next

This is just the beginning.

I'm not trying to become a conventional developer. I'm exploring what it means to build tools that sit at the intersection of education, technology, and human experience.

And now, I know I can actually build them.
